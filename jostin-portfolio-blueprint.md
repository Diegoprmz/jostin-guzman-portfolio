## 1. Project Overview

### Vision
A cutting-edge, immersive 3D web experience for Jostin Guzman, an architect. Users enter a virtual "showroom" where they explore architectural projects through an interactive, WebGL-powered interface. A dramatic loading screen features a door opening with volumetric light illuminating Jostin's logo. The main menu showcases Jostin with a professional black & white studio portrait with parallax depth. From there, users can access individual project rooms—starting with "Complejo Mercado Villa de GPE" and "Comedor Comunitario"—where they interact with rendered scenes, discover points of interest (POI) marked by liquid-glass droplets, view material mood boards, and explore layered 3D annotations.

### Goals
- **Visual Wow Factor**: Establish Jostin as a top-tier, innovative architect through premium visual design
- **Engagement**: Create a memorable, interactive experience that showcases projects at depth
- **Scalability**: Build infrastructure to easily add remaining 5 projects without refactoring
- **Performance**: Maintain 60fps animations even with complex parallax layers and shader effects
- **Mobile Consideration**: Responsive UX (mobile scrolls, desktop supports full interactivity)

### Success Metrics
- Page load under 3s, LCP under 2.5s (Vercel + image optimization)
- Smooth 60fps animations (GPU-accelerated via Three.js + GSAP)
- All POI interactions intuitive and discoverable (visual hierarchy clear)
- First project room fully interactive within 2 weeks of build

## 2. Tech Stack

| Layer | Technology | Why |
|-------|-----------|-----|
| **Framework** | Next.js 15 (App Router) | SSR for SEO, API routes for future CMS integration, Vercel deployment optimization |
| **Language** | TypeScript | Type safety for complex 3D logic, POI managers, shader uniforms |
| **3D/WebGL** | Three.js r168+ | Industry-standard WebGL abstraction, performance, ecosystem (shaders, post-processing) |
| **Graphics** | Custom GLSL shaders | Volumetric lighting (loading door), parallax depth mapping, POI glow/highlight effects |
| **Animations** | GSAP 3 + Framer Motion | Scroll triggers for narrative flow, page transitions, micro-interactions |
| **Styling** | Tailwind CSS v4 + CSS Variables | Dark luxury theme, responsive, dynamic color palettes for lighting changes |
| **UI Components** | shadcn/ui (adapted) | Buttons, modals, drawers, consistent design language |
| **Data** | JSON (local, upgradeable to Sanity/Strapi) | Version control, GitOps workflow, zero database latency for MVP |
| **Fonts** | Inter (headings), JetBrains Mono (technical) | Premium, readable, monospaced for any code-related UI |
| **Hosting** | Vercel | Next.js native, automatic deployments, edge computing for shaders |
| **Package Manager** | pnpm | Fast, efficient, workspace-ready for future expansion |

## 4. Data Model

### Project JSON Structure

All projects stored in `src/data/projects.json`. Each project defines:

```typescript
interface Project {
  id: string;                    // "villa-gpe", "comedor"
  slug: string;                  // URL slug
  title: string;                 // Display name
  year: number;
  location: string;
  category: string;              // "Titulación", "Expo", "Residencial", etc.
  award?: string;                // "1º lugar", "2º lugar", etc.
  description: string;
  longDescription: string;
  client?: string;
  area?: string;                 // Square meters
  team?: string[];
  
  // Visual assets
  hero: {
    image: string;               // /images/projects/{slug}/thumbnails/hero.png
    parallaxLayers: {
      far: string;               // Layer 0 (Z: -100)
      mid: string;               // Layer 1 (Z: -50)
      close: string;             // Layer 2 (Z: 0)
    };
    moodBoard: string;           // /images/projects/{slug}/poi-mood-board.jpg
  };
  
  // Points of Interest
  poi: PointOfInterest[];
  
  // Navigation
  nextProject?: string;          // Slug of next project
  prevProject?: string;          // Slug of previous project
}

interface PointOfInterest {
  id: string;                    // "material-1", "concept-2"
  title: string;                 // "Concrete Finish", "Skylight Design"
  description: string;
  position: {
    x: number;                   // % from left (0-100)
    y: number;                   // % from top (0-100)
  };
  type: "material" | "concept" | "detail" | "structural";
  
  // Info drawer content
  details: {
    heading: string;
    body: string;
    images?: string[];           // Additional reference images
  };
  
  // Mood board refs
  moodBoardItems?: string[];     // ["wood", "concrete", "metal"]
  
  // Interaction config
  radius: number;                // Droplet size (px)
  glowIntensity: number;         // 0.5 - 2.0
}
```

## 6. Frontend Architecture

### Pages & Routes

| Route | Component | Description |
|-------|-----------|-------------|
| `/` | `page.tsx` | Loading screen → auto-redirect to `/menu` on complete |
| `/menu` | `Menu/page.tsx` | Hero with Jostin portrait + 2 project cards, navigation to projects |
| `/project/[slug]` | `ProjectRoom/page.tsx` | Individual project room with parallax, POI, mood board, drawer |

### Component Hierarchy

```
App Layout
├── LoadingScreen (page.tsx → /)
│   ├── DoorOpening (Three.js canvas + volumetric shader)
│   ├── Logo (Animated reveal)
│   └── LoadingBar (Progress)
│
├── Menu (page.tsx → /menu)
│   ├── HeroPortrait (Parallax B&W Jostin)
│   ├── ProjectCard (Links to /project/[slug])
│   └── Navigation (Scroll anchors)
│
└── ProjectRoom (page.tsx → /project/[slug])
    ├── Navigation (Back to /menu, prev/next projects)
    ├── ParallaxCanvas (Three.js, depth layers)
    │   └── LiquidGlassDroplet (POI markers × N)
    ├── MoodBoard (Material references grid)
    └── InfoDrawer (Details on POI click)
```

### State Management

- **Loading State**: Managed by React state in `LoadingScreen`, redirect via Next.js on completion
- **Project Data**: Server-side fetched via `getServerSideProps` equivalent or API route, passed to client
- **POI Interaction State**: Client-side React state (which POI is hovered/selected) + Three.js raycasting for hover detection
- **Scroll Animations**: GSAP ScrollTrigger instances tied to scroll position, no external state manager needed
- **Camera/Parallax**: Three.js scene state, updated each frame via `requestAnimationFrame`

### Spacing & Layout

- **Base Unit**: 4px
- **Spacing Scale**: 4, 8, 12, 16, 24, 32, 48, 64, 96
- **Border Radius**: 8px (default), 12px (cards), 16px (large buttons), 50% (avatars, droplets)
- **Max Content Width**: 1280px
- **Breakpoints**: 
  - `sm`: 640px (mobile)
  - `md`: 768px (tablet)
  - `lg`: 1024px (desktop)
  - `xl`: 1280px (wide desktop)

### Component Style

- **Overall Aesthetic**: Minimal, dark, luxe. Sharp edges with smooth animations. Heavy use of glassmorphism (backdrop blur) and subtle shadows.
- **Buttons**: Solid fill accent color, rounded 8px, no border. On hover: slightly brighter, subtle shadow lift.
- **Cards/Panels**: `background: rgba(255, 255, 255, 0.03)` + `backdrop-filter: blur(16px)` + border `1px solid rgba(255, 255, 255, 0.05)`
- **Shadows**: Subtle depth: `0 4px 16px rgba(0, 0, 0, 0.4)` (default), `0 12px 32px rgba(0, 0, 0, 0.6)` (elevated)
- **Animations**: Smooth ease-in-out, 300-500ms for micro-interactions, 1-2s for narrative animations

### Liquid Glass Droplet (POI)

- **Base**: `background: radial-gradient(135deg, rgba(96, 165, 250, 0.5) 0%, rgba(96, 165, 250, 0.2) 100%)`
- **Border**: `1px solid rgba(96, 165, 250, 0.6)`
- **Backdrop Filter**: `blur(12px)`
- **Size**: 40-60px diameter (configurable per POI)
- **Animation**: Subtle pulse on idle, stronger glow on hover
- **CSS**:
  ```css
  .poi-droplet {
    width: 48px;
    height: 48px;
    border-radius: 50%;
    background: radial-gradient(135deg, rgba(96, 165, 250, 0.5) 0%, rgba(96, 165, 250, 0.2) 100%);
    border: 1px solid rgba(96, 165, 250, 0.6);
    backdrop-filter: blur(12px);
    box-shadow: 0 0 20px rgba(96, 165, 250, 0.4), inset 0 0 20px rgba(255, 255, 255, 0.1);
    cursor: pointer;
    transition: all 300ms ease-out;
  }
  .poi-droplet:hover {
    box-shadow: 0 0 40px rgba(96, 165, 250, 0.8), inset 0 0 30px rgba(255, 255, 255, 0.2);

  }
  ```

Premium 3D interactive portfolio for architect Jostin Guzman, featuring WebGL parallax effects, liquid glass POI interactions, and a dramatic loading screen with volumetric lighting.

## Architecture

### Directory Structure
- `src/app/` — Next.js 15 app router pages (loading, menu, projects)
- `src/components/` — React components (LoadingScreen, Menu, ProjectRoom, POI, etc.)
- `src/lib/` — Three.js setup, shader loaders, POI manager, animation utilities
- `src/hooks/` — Custom hooks (useMouselight, useParallax, useScrollAnimation, useThreeScene)
- `src/data/` — JSON project data, TypeScript schemas
- `public/images/` — Project renders (3 parallax layers per project) + Jostin portrait
- `public/shaders/` — GLSL vertex & fragment shaders (volumetric light, parallax depth, POI glow)

### Key Patterns
1. **Server Components by default** — Fetch project data in layout, pass to client components
2. **Three.js Canvas Wrapper** — Encapsulated in React component with lifecycle management
3. **Shader as Assets** — GLSL files in `public/shaders/`, loaded via `/lib/shaders.ts`
4. **POI Raycasting** — Mouse tracking → detect hover via distance calculation (no physical raycasting)
5. **GSAP ScrollTrigger** — Animations tied to scroll position for narrative flow

### Data Flow
1. **Loading Screen** → Auto-redirect to `/menu` when animation completes
2. **Menu** → Server fetches all projects from `/api/projects`, renders cards
3. **Project Page** → URL slug triggers fetch of `/api/projects/[slug]`, all POI data loaded
4. **POI Interaction** → Client-side state (hovered POI) → render highlight + drawer content
5. **Navigation** → Next/Prev project links, Back to Menu button

## Code Organization Rules

1. **One component per file.** Max 300 lines. Extract sub-components if exceeding.
2. **Path alias:** Use `@/` to import from `src/` (configured in tsconfig.json).
3. **No barrel exports.** Import directly from file: `import { foo } from '@/lib/foo'` not `import { foo } from '@/lib'`.
4. **Server Components by default.** Only add `"use client"` when component needs React state, hooks, or interactivity.
5. **Colocate assets.** Keep component-specific assets (images, shaders) in same folder or `public/` mirroring structure.
6. **Naming:** Components PascalCase, utilities camelCase, constants UPPER_SNAKE_CASE.
